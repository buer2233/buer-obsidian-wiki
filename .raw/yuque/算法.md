## 时间复杂度（Time Complexity）

时间复杂度是用来衡量算法执行时间随输入规模增长的变化趋势。使用大O符号表示。

### O(1) - 常数时间复杂度

执行时间不随输入规模变化。

```python
def get_first(arr):
    return arr[0]  # 无论数组多大，只执行一次

def hash_lookup(hash_table, key):
    return hash_table[key]  # 哈希表查找
```

**常见场景：**
- 数组按索引访问
- 哈希表查找/插入/删除
- 栈的push/pop操作

### O(n) - 线性时间复杂度

执行时间与输入规模成正比。

```python
def linear_search(arr, target):
    for item in arr:
        if item == target:
            return True
    return False

def sum_array(arr):
    total = 0
    for item in arr:
        total += item
    return total
```

**常见场景：**
- 遍历数组/链表
- 线性搜索
- 计数操作

### O(log n) - 对数时间复杂度

每次将问题规模减半。

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

**常见场景：**
- 二分搜索
- 平衡二叉搜索树的查找
- 分治算法

### O(n log n) - 线性对数时间复杂度

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr

    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

**常见场景：**
- 归并排序
- 快速排序（平均情况）
- 堆排序

### O(n^2) - 平方时间复杂度

嵌套循环，每次循环都遍历整个输入。

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr
```

**常见场景：**
- 冒泡排序
- 选择排序
- 插入排序
- 嵌套遍历

### 时间复杂度对比

| 复杂度 | 名称 | 10个元素 | 100个元素 | 1000个元素 |
|-------|------|---------|----------|-----------|
| O(1) | 常数 | 1 | 1 | 1 |
| O(log n) | 对数 | 3 | 7 | 10 |
| O(n) | 线性 | 10 | 100 | 1000 |
| O(n log n) | 线性对数 | 30 | 700 | 10000 |
| O(n^2) | 平方 | 100 | 10000 | 1000000 |

## 空间复杂度（Space Complexity）

空间复杂度衡量算法所需额外存储空间随输入规模增长的变化趋势。

### O(1) - 常数空间

```python
def sum_array(arr):
    total = 0
    for item in arr:
        total += item
    return total
```

### O(n) - 线性空间

```python
def create_copy(arr):
    return arr[:]  # 创建一个等大的副本
```

### O(n^2) - 平方空间

```python
def create_matrix(n):
    return [[0] * n for _ in range(n)]
```

## 大O符号（Big O Notation）

大O符号用于描述算法的渐进上界，表示算法在最坏情况下的时间或空间复杂度。

### 规则

1. **常数可以忽略**：O(2n) = O(n)
2. **低阶项可以忽略**：O(n^2 + n) = O(n^2)
3. **系数可以忽略**：O(3n^2) = O(n^2)

### 常见复杂度排序

```
O(1) < O(log n) < O(n) < O(n log n) < O(n^2) < O(n^3) < O(2^n) < O(n!)
```

### 最好、最坏、平均情况

```python
# 最好情况：O(1) - 第一个元素就是目标
# 最坏情况：O(n) - 目标在最后或不存在
# 平均情况：O(n/2) = O(n)
def linear_search(arr, target):
    for i, item in enumerate(arr):
        if item == target:
            return i
    return -1
```

### 摊还分析（Amortized Analysis）

考虑一系列操作的平均时间复杂度，而不是单个操作的最坏情况。

例如：动态数组的扩容操作
- 单次扩容：O(n)
- n次插入的摊还复杂度：O(1)
