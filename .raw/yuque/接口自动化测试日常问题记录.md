2.urlencoded接口请求

```python
# 示例1
def ebComs_projectResourceBoard_previewData(self, ETEAMSID, page_id, com_id, config, return_df=False, **kwargs):
    """EB页面 - 模块组件 - 项目管理 - 资源负荷看板 - 设计器页面预览数据"""
    url = "https://{0}/api/ebuilder/coms/projectResourceBoard/previewData".format(self.base_url)
    payload = {
        "pageId": page_id,
        "compId": com_id,
        "config": json.dumps(config),
        "pageSize": '20',
        "pageNo": '1',
        "timeStamp": self.generate_timeStamp()
    }
    error_msg = kwargs.pop("error_msg", "EB页面 - 模块组件 - 项目管理 - 资源负荷看板 - 设计器页面预览数据")
    response_code = kwargs.pop("response_code", 200)
    payload.update(kwargs)
    headers = {
        'Cookie': "ETEAMSID=" + ETEAMSID + "",
        'Content-Type': 'application/x-www-form-urlencoded',
        'timeZoneOffset': '-480'
    }
    res = requests.request("POST", url, headers=headers, data=urlencode(payload))
    assert res.status_code == 200, f"{error_msg}-接口<{url}>报错:{res.status_code},reason:{res.reason},text:{res.text}"
    response = res.json()
    assert response.get("code") == response_code, f"{error_msg}-接口<{url}>报错-{response}"
    if return_df:
        assert response.get("data"), f"{error_msg}-接口<{url}>错误:{response}"
        return DataFrame(response["data"]["list"])
    return response
```

17. pytest8.x版本不兼容历史项目代码中conftest写法的兼容性问题

问题表现：从python3.7替换到Python3.12后(实际升级造成问题的原因是pytest从7.0版本升级了8.0版本)。执行UI自动化测试用例，报错无法获取到根目录conftest.py里面的全局fixture方法。

问题原因：pytest 8.x 以上版本的conftest.py加载规则重大变更：当测试用例所在目录存在非空的conftest.py时，pytest 8.x 会默认终止向上递归查找更高层级的 conftest.py 文件。

解决方案：在测试用例的子目录conftest.py文件最上方强制引用上层conftest.py内的内容。
