## 二、AI辅助测试工具能做什么

| 场景 | AI作用 |

```text

```

## 三、AI生成测试用例如何保证质量

质量保障流程：

```text
输入标准化 -> 上下文检索 -> 模板/规则约束 -> AI生成 -> 人工审核 -> 自动执行 -> 失败修复 -> 结果沉淀
```

具体措施：
- 输入明确：接口文档、cURL、抓包、已有用例、源码。
- 输出模板固定：接口方法、pytest用例、断言说明。
- 规则约束：写入位置、命名、断言要求、禁止跳过失败。
- 执行验证：必须跑pytest。
- 人工审核：关键业务断言。
- 数据统计：通过率、修改率、节省时间。

## 四、AI自动写接口用例流程

结合你的真实项目：

```text
1. 输入来源：mitmproxy抓包、cURL、已有用例、Java Controller源码。
2. AI读取项目规则和接口索引。
3. AI生成接口方法和pytest用例。
4. 执行最小范围pytest。
5. 如果失败，读取日志和响应修复。
6. 检查Git diff。
7. 人工确认业务断言。
8. 记录提效数据。
```

四种工作流：
- 抓包驱动。
- 参考已有用例。
- cURL手工。
- Java Controller源码参考。

## 五、最大难点和解决方案

| 难点 | 表现 | 解决 |
|------|------|------|
| 上下文质量差 | AI不了解项目 | 接口索引、已有用例、规则文件 |
| 输出不稳定 | 每次结构不一样 | 模板化、固定格式 |
| 业务理解不足 | 断言浅 | 人工确认关键断言 |
| 执行不闭环 | 只生成不验证 | 强制pytest |
| 模型绕过失败 | 跳过失败用例 | 禁止skip，检查diff |
| 推广困难 | 团队不会用 | 写Skill文档和培训 |
| 安全边界 | 乱改文件/访问敏感数据 | 前置门禁、工作目录限制 |

## 六、如何证明AI项目价值

指标：
- 接口用例产出数量。
- 接口方法产出数量。
- 单用例耗时。
- pytest一次通过率。
- 人工修改率。
- 缺陷发现数。
- 回归时间缩短。
- 团队使用次数。
- 失败反馈闭环次数。

```text
未使用AI前5周：129接口方法 + 198接口用例 = 327
使用AI后5周：96接口方法 + 383接口用例 = 479
接口用例编写效率提升93.43%，综合提升46.48%
```

表达方式：
```text
我不是只说效率提升，而是保留了AI使用前后同周期产出数据，通过接口用例数量、接口方法数量和总工作量对比，证明AI确实提高了接口用例编写效率。
```

## 七、Prompt工程、上下文工程、Harness工程

| 概念 | 关注点 |
|------|--------|
| Prompt工程 | 怎么写指令让单次输出更好 |
| 上下文工程 | 给模型什么资料 |
| Harness工程 | 约束、工具、门禁、反馈闭环 |
| Loop工程 | 失败后如何基于反馈继续修复 |

你的项目体现：
- 前置门禁：必须填写写入位置。
- 渐进式披露：按工作流读取子文档。
- 工具：抓包、接口索引、pytest、Git diff。
- 反馈：pytest失败日志回传修复。
- 约束：禁止跳过失败，人工确认断言。

面试展开：

```text
Prompt工程解决“怎么问”，上下文工程解决“给什么资料”，harness工程解决“怎么约束AI行动”，loop工程解决“失败后怎么继续修复”。我的AI+接口自动化SKILL正好覆盖这四层：用固定Prompt约束输出，用接口索引和已有用例补上下文，用pytest和Git diff做门禁，用失败日志回传形成修复循环。
```

多LLM集成表达：

```text
我在AI+UI/browser-use方向集成过OpenAI、Claude、DeepSeek、硅基流动、阿里云百炼等模型。多LLM不是简单换API key，还要抽象模型配置、统一输入输出、处理不同模型的工具调用能力、超时重试、成本和效果差异，并用同一批任务集做回归对比。
```

## 八、结合个人项目深挖

### 1. AI+接口自动化SKILL

```text
这是我最核心的AI测试落地项目。它支持4种工作流：抓包驱动、参考已有用例、cURL手工、Java Controller源码参考。AI先读取项目规则和上下文，再生成接口方法和pytest用例，之后执行最小范围pytest，失败就读取日志修复，最后检查Git diff并由人工确认关键业务断言。
```

可背诵亮点：
- 不是单次Prompt，而是完整工程闭环。
- 不是只生成代码，而是生成、执行、修复、确认。
- 不是只说提效，而是用前后5周数据证明：接口用例编写效率提升93.43%，综合效率提升46.48%。
- 不是个人玩具，已经在公司内部推广使用。

### 2. AI+UI browser-use

```text
browser-use本质是浏览器Agent。LLM观察页面状态，理解自然语言任务，规划下一步动作，再通过Playwright执行点击、输入、跳转等操作。我二开时重点关注多LLM集成、任务执行链路、日志轨迹和测试平台/Jenkins集成，让自然语言场景可以进入自动化测试流程。
```

测试关注：
- 元素识别是否准确。
- 页面变化后是否能恢复。
- 动作轨迹是否可回放。
- 高风险动作是否被限制。
- 不同LLM在同一任务集上的成功率和稳定性差异。

```text

```

