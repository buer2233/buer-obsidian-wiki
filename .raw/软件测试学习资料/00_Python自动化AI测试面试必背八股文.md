## 二、Python测试开发必背八股

### 1. Python语言基础

| 面试题 | 必背要点 |
|--------|----------|
| Python可变对象和不可变对象区别？ | list/dict/set可变，int/str/tuple不可变；函数传参是对象引用传递；默认参数不要用可变对象 |
| 深拷贝和浅拷贝区别？ | 浅拷贝只复制外层对象，深拷贝递归复制内部对象；接口测试数据模板常用 |
| `*args` 和 `**kwargs` 是什么？ | 可变位置参数和可变关键字参数；常用于封装通用请求方法 |
| 装饰器原理是什么？ | 函数作为一等对象；闭包；不修改原函数代码增强行为；测试中用于日志、重试、计时 |
| 生成器 `yield` 怎么执行？ | 遇到 yield 暂停并返回值；下一次 next 从暂停处继续；适合大数据、流式处理 |
| 上下文管理器是什么？ | `with` 调用 `__enter__` / `__exit__`；用于文件、数据库连接、资源释放 |
| 列表推导式和生成器表达式区别？ | 列表推导式一次生成列表；生成器惰性计算，节省内存 |
| GIL是什么？ | CPython全局解释器锁；CPU密集型多线程受限；IO密集型仍可受益 |
| 多线程、多进程、协程区别？ | 线程适合IO并发；进程适合CPU密集；协程适合高并发IO |
| 异常处理怎么设计？ | 捕获具体异常；保留日志和堆栈；接口自动化中区分请求异常、断言失败、环境异常 |

### 2. pytest必背

| 面试题 | 必背要点 |
|--------|----------|
| pytest和unittest区别？ | pytest语法简洁、fixture强大、插件生态丰富、参数化方便；unittest是标准库 |
| fixture是什么？ | 测试前后置资源管理；可用于登录、造数据、清理数据、浏览器启动 |
| fixture有哪些scope？ | function、class、module、package、session |
| fixture和setup/teardown区别？ | fixture更灵活，可复用、可依赖、可参数化 |
| `conftest.py`有什么作用？ | pytest自动发现；存放公共fixture、hook、插件配置 |
| 参数化怎么做？ | `@pytest.mark.parametrize`；覆盖多组输入输出 |
| fixture参数化和parametrize区别？ | fixture params偏测试资源组合；parametrize偏测试数据组合 |
| indirect参数化是什么？ | 将参数传给fixture，由fixture处理复杂前置逻辑 |
| pytest hook是什么？ | 扩展pytest生命周期，如收集用例、修改报告、失败截图 |
| 如何生成测试报告？ | pytest-html、Allure、自定义hook；报告需包含用例、日志、截图、失败原因 |

### 3. Python测试开发框架设计

| 面试题 | 必背要点 |
|--------|----------|
| 你怎么设计接口自动化框架？ | 分层：config、api/client、case、data、assert、utils、report、log、CI |
| 数据驱动怎么做？ | yaml/json/excel/db；测试数据与测试逻辑分离 |
| 关键字驱动是什么？ | 将操作封装成关键字，降低用例维护成本 |
| 如何处理测试环境配置？ | env配置隔离；base_url、账号、token、数据库、开关 |
| 如何处理测试数据？ | 前置构造、接口造数、数据库造数、清理策略、数据隔离 |
| 如何处理接口依赖？ | 提取上个接口返回值、上下文变量、fixture链式依赖 |
| 如何保证用例稳定？ | 数据隔离、重试、显式等待、失败截图/日志、环境检查 |
| 如何接入CI/CD？ | Jenkins流水线拉代码、装依赖、执行pytest、生成报告、通知 |

## 三、自动化测试必背八股

### 1. 测试理论与用例设计

| 面试题 | 必背要点 |
|--------|----------|
| 测试流程是什么？ | 需求分析、测试计划、用例设计、执行、缺陷跟踪、回归、报告 |
| 黑盒测试方法有哪些？ | 等价类、边界值、判定表、因果图、场景法、错误推测 |
| 等价类怎么设计？ | 有效等价类、无效等价类；每类选代表值 |
| 边界值怎么设计？ | 最小值、最大值、临界值、刚好越界值 |
| 测试金字塔是什么？ | 单元测试多、接口/服务测试适中、UI测试少；越上层越慢越不稳定 |
| 冒烟测试和回归测试区别？ | 冒烟确认主流程可用；回归确认改动未破坏旧功能 |
| Bug生命周期是什么？ | New、Assigned、Fixed、Retest、Closed、Reopen、Rejected |
| 你如何判断一个需求可测？ | 输入输出明确、验收标准明确、异常场景明确、数据规则明确 |

### 2. 接口自动化必背

| 面试题 | 必背要点 |
|--------|----------|
| 接口测试测什么？ | 请求方法、参数、鉴权、业务规则、响应结构、状态码、数据库、日志 |
| 服务端下单接口怎么测？ | 业务链路、参数、库存、金额、优惠、幂等、并发、数据库、异常场景 |
| GET/POST/PUT/PATCH/DELETE区别？ | GET查询；POST创建/提交；PUT整体更新；PATCH局部更新；DELETE删除 |
| 401和403区别？ | 401未认证；403已认证但无权限 |
| 4xx和5xx区别？ | 4xx请求侧问题；5xx服务端/网关/依赖问题 |
| 业务错误码和HTTP状态码区别？ | HTTP表示协议层；业务错误码表示业务层失败原因 |
| 接口断言断什么？ | 状态码、响应字段、业务码、数据类型、数据库、日志、副作用 |
| token/cookie/session怎么处理？ | 登录接口获取；统一保存；请求头注入；过期刷新 |
| 接口关联怎么处理？ | 上下文变量、fixture依赖、提取响应字段 |
| Mock和Stub是什么？ | Mock模拟依赖服务；Stub固定返回；用于隔离外部依赖 |
| 接口幂等性怎么测？ | 重复提交同一请求，验证不会重复扣款/扣库存/生成多单 |
| 接口自动化失败怎么排查？ | 请求参数、环境配置、测试数据、接口日志、数据库、依赖服务、网络 |

### 3. UI自动化必背

| 面试题 | 必背要点 |
|--------|----------|
| Selenium和Playwright区别？ | Selenium生态成熟；Playwright现代化、自动等待强、上下文隔离好 |
| 显式等待和隐式等待区别？ | 显式等待针对具体条件；隐式等待全局查找元素等待 |
| Playwright自动等待是什么？ | 执行动作前自动等待元素可见、稳定、可接收事件、可用 |
| Locator优势是什么？ | 延迟定位、稳定、可组合、推荐使用 |
| PO模式是什么？ | 页面对象封装元素和操作，测试用例只表达业务流程 |
| 如何处理iframe？ | 切换frame或使用frame locator |
| 如何处理弹窗/上传/下载？ | 监听事件、文件路径、上下文管理 |
| UI自动化不稳定怎么解决？ | 稳定定位、显式等待、数据隔离、减少强依赖、失败截图、重试 |
| 哪些场景不适合UI自动化？ | 频繁变化页面、一次性活动、极复杂视觉校验、成本高于收益场景 |

### 4. 性能测试必背

| 面试题 | 必背要点 |
|--------|----------|
| 性能测试有哪些类型？ | 基准、负载、压力、稳定性、容量、峰值、并发测试 |
| TPS/QPS/响应时间/并发用户是什么？ | TPS事务每秒；QPS请求每秒；响应时间分平均/P90/P95/P99；并发是同时在线/同时请求 |
| JMeter线程组参数怎么理解？ | 线程数、Ramp-up、循环次数、调度器 |
| JMeter Test Plan有哪些组件？ | Thread Group、Sampler、Controller、Listener、Timer、Assertion、Config Element |
| 如何判断性能瓶颈？ | 看应用CPU/内存/线程、数据库慢SQL/锁、网络、缓存、队列、第三方依赖 |
| TPS下降原因有哪些？ | 数据库瓶颈、连接池耗尽、锁竞争、GC、CPU满、网络延迟、依赖服务慢 |
| 性能报告怎么写？ | 场景、环境、数据量、并发模型、指标、瓶颈、结论、优化建议 |
| Locust优势是什么？ | Python脚本化、协程并发、适合复杂业务流程 |

### 5. Linux / MySQL / Docker / Jenkins必背

## 四、AI测试必背八股

### 1. AI测试基础

| 面试题 | 必背要点 |
|--------|----------|
| AI测试和传统测试区别？ | 传统系统输出确定；AI系统输出概率性、非确定性，需要评估集和指标 |
| 大模型应用怎么测试？ | 功能、准确性、相关性、幻觉、安全、鲁棒性、性能、成本、可观测性 |
| 什么是幻觉？ | 模型生成不符合事实或上下文的内容；需要事实校验、引用校验、人工/自动评估 |
| Prompt怎么测试？ | 正常、边界、歧义、恶意注入、越权、敏感信息、稳定性、多轮上下文 |
| AI测试指标有哪些？ | 准确率、相关性、忠实性、召回率、拒答率、幻觉率、延迟、成本、成功率 |
| 如何构建AI评测集？ | 收集真实问题、标注期望答案、覆盖场景/边界/风险、持续维护版本 |
| AI缺陷怎么定位？ | 分层定位：Prompt、检索、工具调用、模型、数据、上下文、后处理、权限 |

### 2. RAG测试必背

| 面试题 | 必背要点 |
|--------|----------|
| RAG系统怎么测试？ | 检索质量、上下文质量、答案质量、引用正确性、端到端效果 |
| RAG常见质量问题？ | 检索不到、检索错、上下文冲突、答案幻觉、引用错误、过期知识 |
| RAG指标有哪些？ | context precision、context recall、answer relevancy、faithfulness |
| 如何降低幻觉？ | 检索增强、引用约束、拒答策略、答案校验、评估集回归 |
| 如何测试知识库更新？ | 新文档可检索、旧文档失效、版本隔离、权限过滤、增量索引 |
| 如何测试权限隔离？ | 不同用户只检索有权限文档，防止越权泄露 |

### 3. Agent测试必背

| 面试题 | 必背要点 |
|--------|----------|
| AI Agent怎么测试？ | 任务理解、规划、工具选择、工具参数、执行结果、异常恢复、日志审计 |
| 工具调用怎么校验？ | 是否调用正确工具、参数是否正确、权限是否正确、失败是否重试 |
| Agent常见风险？ | 错误工具调用、无限循环、越权操作、成本失控、上下文污染 |
| Agent自动化测试怎么做？ | 构造任务集，记录轨迹，校验步骤、输出、工具调用和最终结果 |
| browser-use类项目怎么讲？ | LLM理解任务 -> 规划动作 -> Playwright执行 -> 观察反馈 -> 循环直到完成 |

### 4. LLM安全测试必背

| 面试题 | 必背要点 |
|--------|----------|
| Prompt Injection是什么？ | 恶意输入覆盖系统指令或诱导模型越权 |
| Sensitive Information Disclosure是什么？ | 模型泄露敏感信息、密钥、内部数据 |
| Insecure Output Handling是什么？ | 未校验模型输出导致XSS、SQL注入、命令执行等 |
| Model DoS是什么？ | 超长输入、复杂任务或循环导致资源耗尽 |
| Supply Chain风险是什么？ | 模型、插件、数据、第三方依赖被污染 |
| AI安全测试怎么做？ | 构造攻击样本、越权场景、敏感词、长上下文、多轮诱导、输出过滤校验 |

2. DRF后端负责什么？
3. 用例、任务、报告、环境、数据如何建模？
4. 如何触发自动化执行？
5. 如何和Jenkins集成？
6. 如何做权限、日志、报告和失败追踪？

### 4. 疑难点案例

至少准备3个：

| 案例 | 表达结构 |
|------|----------|
| 接口自动化偶发失败 | 现象 -> 排查参数/数据/日志/数据库 -> 根因 -> 数据隔离/初始化 -> 稳定性提升 |
| AI生成用例质量不稳定 | 现象 -> 分析Prompt/上下文/模板 -> 增加规则和校验 -> 执行验证闭环 |
| UI自动化脚本不稳定 | 现象 -> 定位等待/元素/数据/环境 -> 改Locator/等待/截图/重试 |
