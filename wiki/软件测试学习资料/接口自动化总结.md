## 2.TCP/IP协议家族

### TCP（Transmission Control Protocol 传输控制协议）是一种面向连接的、可靠的、基于字节流的传输层通信协议

### TCP协议头

#### 端口号
- 区别同一电脑的不同应用,加上IP地址就可以唯一标记互联网的所有应用
- 范围
- 0~65535
- 0~1023
- 系统默认使用
- 常见应用,占用的端口
- 3306
- MySQL
- 80
- http协议
- 443
- HTTPS协议
- 8080
- Tomcat/Apache
- 27017
- MongoDB
- 6379
- Redis
- 8086
- influxDB
- 8888
- 抓包工具:fiddler
- jmeter的HTTP代理服务器:用于录制web或移动端的操作
- 查看端口是否占用
- window
- netstat -ano | findstr 端口号
- linux
- netstat -pan | grep 端口号

#### 序列号seq
- 当第一次发送请求时,会随机生成序列
- 在第二次请求时,在之前序列号的基础上加1

#### 标记
- SYN
- 当SYN=1时,表示和对方建立连接
- ACK
- 但ACK=1时,表示回复收到对象的请求
- FIN
- 但FIN=1时,表示和对方断开连接

### TCP的三次握手

#### 第一次握手
- 客户端向服务器发送建立连接的请求
- SYN=1, seq=x

#### 第二次握手
- 服务器回复客户端,同意建立连接,并建立连接
- ACK=1, ack=x+1, SYN=1, seq=y

#### 第三次握手
- 客户端回复服务器,收到对方请求
- ACK=1, ack=y+1, seq=x+1

### TCP的四次挥手

#### 第一次挥手
- 客户端向服务器发送断开连接的请求
- FIN=1, seq=x

#### 第二次挥手
- 服务器回复客户端,同意断开连接.但此时数据可能还未传输完毕,所有不会马上执行断开连接的操作
- ACK=1, ack=x+1, seq=y

#### 第三次挥手
- 数据传输完毕之后,服务器向客户端发送断开连接的请求
- FIN=1, ack=x+1, seq=z

#### 第四次挥手
- 客户端回复收到服务器断开连接的请求
- ACK=1, ack=z+1, seq=x+1

## 3.HTTP协议

### 发送http请求之前,需要先建立TCP连接,也就是TCP的三次握手

### 1.HTTP请求地址_URL

#### 示例
- http://localhost:8080/ecshop/admin/index.php?username=root&password=root123456

#### http://
- 协议部分

#### localhost
- IP地址/域名
- 域名是通过DNS转换为IP地址

#### 8080
- 端口号

#### /
- 根目录

#### ecshop/admin/
- 文件路径

#### index.php
- 文件名

#### ?username=root&password=root123456
- get请求的请求参数
- 使用 ? 号分割地址和参数
- 使用 & 号连接多个请求参数

### 2.HTTP请求_Request

#### 示例
- GET / HTTP/1.1    # 请求行
Host: www.baidu.com
User-Agent: curl/7.55.1   # 请求头
Accept: */*
# 空行
请求体(请求数据)   #请求体

#### 请求行
- 请求方式
- get
- 请求地址
- /
- 协议版本
- HTTP/1.1

#### 请求头
- host
- 主机名
- user-agent
- 客户端信息
- accept
- 可接受的数据类型
- cookie/token
- Content-Type
- 请求数据的类型

#### 空行
- 用于分割请求头和请求参数

#### 请求体(请求参数)
- post请求的请求参数
- x-www-form-urlencoded
- form-data
- JSON
- binary

### 4.HTTP请求方式_method

- 200

#### post
- 先服务器新增资源
- 201

#### delete
- 删除服务器资源
- 204

#### 修改服务器服务器资源
- 201

#### head

#### options
- 检测请求方式

### 5.HTTP状态码_status code

#### 2XX: 响应成功
- 200
- 响应成功
- 201
- 请求成功且执行实现,而且有一个新的资源已经依据请求的需要建立
- 202
- 服务器接受请求,但未处理

#### 3XX: 重定向
- 301
- 永久重定向
- 302
- 临时重定向

#### 4XX: 客户端问题
- 401
- 未授权,一般是未登录
- 403
- 请求被拒绝,无请求权限
- 404
- 请求资源不存在
- 可能是请求参数填写错误
- 也可能是服务器无请求对应的资源

#### 5XX: 服务器问题
- 500
- 服务器发生未知的错误,导致无法处理请求
- 501
- 服务器暂时不支持当前的功能
- 503
- 因为临时的负载或维护造成的暂时性的服务器错误,过一段时间就可使用
