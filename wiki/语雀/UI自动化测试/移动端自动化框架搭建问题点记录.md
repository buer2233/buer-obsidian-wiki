# 移动端自动化框架搭建问题点记录

## 问题1: Appium Server 连接失败

### 问题描述

Python客户端无法连接到Appium Server，报错 `ConnectionRefusedError`。

### 解决方案

```python
# 检查Appium Server是否启动
# 终端执行:
appium --address 127.0.0.1 --port 4723

# 检查端口是否被占用
# Windows
netstat -ano | findstr 4723
# Linux/Mac
lsof -i :4723

# Python连接配置
from appium import webdriver

desired_caps = {
    "platformName": "Android",
    "appium:deviceName": "emulator-5554",
    "appium:appPackage": "com.example.app",
    "appium:appActivity": ".MainActivity",
    "appium:automationName": "UiAutomator2"
}

driver = webdriver.Remote(
    "http://127.0.0.1:4723",
    desired_caps
)
```

## 问题2: 设备无法识别

### 问题描述

`adb devices` 显示设备为 `unauthorized` 或无设备。

### 解决方案

```bash
# 1. 检查USB调试是否开启
# 设置 -> 开发者选项 -> USB调试

# 2. 重启adb服务
adb kill-server
adb start-server

# 3. 授权弹窗
# 设备上会弹出授权对话框，点击"允许"

# 4. 检查USB驱动(Windows)
# 安装对应手机品牌的USB驱动

# 5. 使用WiFi连接(替代USB)
adb tcpip 5555
adb connect 192.168.1.100:5555

# 6. 检查USB线是否支持数据传输
# 有些线只能充电
```

## 问题3: UiAutomator2 安装失败

### 问题描述

Appium启动时 UiAutomator2 驱动安装失败。

### 解决方案

```bash
# 方案1: 手动安装驱动
appium driver install uiautomator2

# 方案2: 清除缓存重新安装
rm -rf ~/.appium/node_modules/appium-uiautomator2-driver
appium driver install uiautomator2

# 方案3: 检查SDK路径
echo $ANDROID_HOME
# 确保路径正确且包含platform-tools

# 方案4: 检查Java版本
java -version
# 推荐JDK 8或11

# 方案5: 设置capabilities跳过安装
desired_caps["appium:skipServerInstallation"] = True
```

### 问题描述

Appium无法安装被测应用到设备上。

### 解决方案

```python
# 方案1: 使用绝对路径
desired_caps["appium:app"] = "/full/path/to/app.apk"

# 方案2: 检查APK文件完整性
import os
apk_path = "/path/to/app.apk"
print(f"文件大小: {os.path.getsize(apk_path)} bytes")

# 方案3: 手动安装后跳过
import subprocess
subprocess.run(["adb", "-s", "设备ID", "install", "-r", "app.apk"])

desired_caps["appium:noReset"] = True
desired_caps["appium:fullReset"] = False

# 方案4: 检查存储空间
# adb shell df -h

# 方案5: 卸载旧版本重装
```

## 问题5: 元素定位超时

### 问题描述

页面元素加载慢，导致定位超时。

### 解决方案

```python
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from appium.webdriver.common.appiumby import AppiumBy

# 方案1: 增加等待时间
element = WebDriverWait(driver, 30).until(
    EC.presence_of_element_located(
        (AppiumBy.ID, "com.example:id/button")
    )
)

# 方案2: 自定义等待条件
def wait_for_element(driver, locator, timeout=30):
    """等待元素出现并可点击"""
    try:
        element = WebDriverWait(driver, timeout).until(
            EC.element_to_be_clickable(locator)
        )
        return element
    except TimeoutException:
        # 截图保存现场
        driver.save_screenshot("timeout_error.png")
        raise

# 方案3: 使用poll_frequency调整轮询间隔
element = WebDriverWait(
    driver, 30, poll_frequency=0.5
).until(
    EC.presence_of_element_located(
        (AppiumBy.XPATH, "//android.widget.Button[@text='确定']")
    )
)

# 方案4: 滚动查找元素
def scroll_find_element(driver, text):
    """滚动查找包含指定文本的元素"""
    script = 'new UiScrollable(new UiSelector().scrollable(true))' \
             '.scrollIntoView(new UiSelector().text("' + text + '"))'
    return driver.find_element(AppiumBy.ANDROID_UIAUTOMATOR, script)
```

## 问题6: 真机性能问题

### 问题描述

真机测试时响应慢、操作卡顿。

### 解决方案

```python
# 1. 减少不必要的等待
desired_caps["appium:newCommandTimeout"] = 60

# 2. 禁用不需要的功能
desired_caps["appium:noReset"] = True
desired_caps["appium:skipDeviceInitialization"] = True
desired_caps["appium:skipUnlock"] = True

# 3. 使用UiAutomator2的fastReset
desired_caps["appium:fastReset"] = True

# 4. 关闭动画(开发者选项)
# 窗口动画缩放 -> 关闭
# 过渡动画缩放 -> 关闭
# 动画程序时长缩放 -> 关闭

# 5. 使用adb命令关闭动画
adb shell settings put global window_animation_scale 0
adb shell settings put global transition_animation_scale 0
adb shell settings put global animator_duration_scale 0
```

## 问题7: 中文输入问题

### 问题描述

输入中文时出现乱码或无法输入。

### 解决方案

```python
# 方案1: 使用Unicode输入
element.send_keys("测试文本")

# 方案2: 使用adb命令输入中文
import subprocess
subprocess.run(["adb", "shell", "input", "text", "测试"])

# 方案3: 安装Unicode输入法
desired_caps["appium:unicodeKeyboard"] = True
desired_caps["appium:resetKeyboard"] = True

# 方案4: 使用剪贴板
from appium.webdriver.clipboard_content import ClipboardContentType
driver.set_clipboard("测试文本".encode('utf-8'), ClipboardContentType.PLAINTEXT)
element.send_keys(Keys.CONTROL, 'v')
```

## 问题8: 多设备并行测试

### 问题描述

需要同时在多台设备上运行测试。

### 解决方案

```python
# 1. 获取所有连接的设备
import subprocess

def get_devices():
    result = subprocess.run(
        ["adb", "devices"],
        capture_output=True, text=True
    )
    devices = []
    for line in result.stdout.strip().split('\n')[1:]:
        if '\tdevice' in line:
            devices.append(line.split('\t')[0])
    return devices

# 2. 为每台设备创建独立的driver
def create_driver(device_id):
    caps = {
        "platformName": "Android",
        "appium:deviceName": device_id,
        "appium:appPackage": "com.example.app",
        "appium:appActivity": ".MainActivity",
        "appium:automationName": "UiAutomator2",
        "appium:noReset": True
    }
    return webdriver.Remote("http://127.0.0.1:4723", caps)

# 3. 使用多线程并行执行
import threading

def run_test(device_id):
    driver = create_driver(device_id)
    # 执行测试...
    driver.quit()

devices = get_devices()
threads = []
for device in devices:
    t = threading.Thread(target=run_test, args=(device,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()
```
