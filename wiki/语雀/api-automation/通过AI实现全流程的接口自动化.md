## 为什么是扫描前端仓库

默认前端在用的接口都是提供给外部用户会用到的接口。后端接口可能存在废弃未使用，或是内部调用的接口等情况。

## 全流程自动化提示词

```
/using-superpowers
你需要按照下面的要求，执行一个长任务，执行任务前请认真进行规划，并严格按照规划执行这个长任务。在完成下面每个步骤任务的同时，你需要把每个步骤的执行经验，通过/skill-creator技能都提炼为一个专用的SKILL，存放在当前项目的.claude\skills下，以便后续可以直接复用使用。

● 你本次任务需要扫描的前端仓库目录：[前端仓库路径]

● 下面扫描和分析后的产物，全部放置在该目录：[产物输出目录]

1. 扫描指定目录下的整个前端代码仓库生成详细的接口文档，需要包含详细的接口URL、method、所有请求参数信息、Content-Type等信息。生成接口文档。

2. 拿到接口文档的每一个接口，挨个去前端项目中查询接口的调用链路，最后汇总在同一链路上的接口，生成不同功能调用链路的文档。

3. 通过调用链路文档，设计并生成功能测试用例，功能测试用例的每个步骤都要绑定对应的接口。

4. 按照绑定了接口的功能测试用例和接口文档，使用生成接口用例的SKILL批量生成接口自动化测试用例。

其它注意事项：

1. 需要注意不同Content-Type类型接口的请求参数是有差异的，你需要根据不同的Content-Type类型接口，提取不同的请求参数，最终生成一份能够按照参数就请求通接口的接口文档。注意请求参数必须完整，不能有任何缺失。

2. 接口用例生成，你需要使用技能/api-test-E10，但是本次你无需参考任何项目内的已覆盖接口对应的接口方法。所有接口全部新生成接口方法，并新增接口放置在指定文件的末尾追加。

3. 测试框架内容参考测试技能/api-test-E10
```

## 关键经验

1. **扫描前端仓库的优势：** 前端仓库中的接口是实际使用的接口，而后端可能存在废弃或内部调用的接口

2. **全流程自动化流程：**
   - 扫描前端代码 → 生成接口文档
   - 分析接口调用链路 → 生成功能调用链路文档
   - 设计功能测试用例 → 绑定接口
   - 批量生成接口自动化测试用例

3. **SKILL提炼：** 每个步骤的执行经验都应该提炼为专用的SKILL，以便后续复用
