# 性能测试基础

## 1. 性能测试的目的与价值

### 为什么需要性能测试

| 目的 | 说明 |
|------|------|
| 评估系统能力 | 确定系统最大承载能力 |
| 发现性能瓶颈 | 找出系统薄弱环节 |
| 验证稳定性 | 确保系统长时间稳定运行 |
| 优化依据 | 为性能优化提供数据支撑 |
| 容量规划 | 指导硬件资源采购 |
| 风险预估 | 上线前评估风险 |

### 性能测试类型

```
性能测试
├── 负载测试 (Load Testing)
│   ├── 验证系统在预期负载下的表现
│   └── 找出系统性能拐点
├── 压力测试 (Stress Testing)
│   ├── 验证系统在极限负载下的表现
│   └── 找出系统崩溃点
├── 稳定性测试 (Soak Testing)
│   ├── 验证系统长时间运行的稳定性
│   └── 发现内存泄漏等问题
├── 并发测试 (Concurrency Testing)
│   ├── 验证系统并发处理能力
│   └── 发现死锁、资源竞争问题
└── 容量测试 (Capacity Testing)
    ├── 确定系统最大承载能力
    └── 指导容量规划
```

### 关键性能指标

| 指标 | 英文 | 说明 |
|------|------|------|
| 响应时间 | Response Time | 请求到响应的时间 |
| 吞吐量 | Throughput | 单位时间处理的请求数 |
| 并发用户数 | Concurrent Users | 同时在线的用户数 |
| TPS | Transactions Per Second | 每秒事务数 |
| QPS | Queries Per Second | 每秒查询数 |
| 错误率 | Error Rate | 失败请求占比 |
| 资源利用率 | Resource Usage | CPU、内存、磁盘、网络 |

## 2. 性能测试标准流程

### 测试流程

```
1. 需求分析
   ├── 明确测试目标
   ├── 确定性能指标
   └── 了解系统架构

2. 测试计划
   ├── 制定测试方案
   ├── 确定测试环境
   └── 准备测试数据

3. 环境准备
   ├── 搭建测试环境
   ├── 部署被测系统
   └── 配置监控工具

4. 脚本开发
   ├── 编写测试脚本
   ├── 参数化数据
   └── 添加断言

5. 测试执行
   ├── 预热运行
   ├── 正式测试
   └── 监控记录

6. 结果分析
   ├── 数据整理
   ├── 瓶颈分析
   └── 报告编写

7. 调优验证
   ├── 性能调优
   ├── 回归测试
   └── 对比分析
```

### 性能指标参考

| 指标 | 优秀 | 良好 | 及格 | 不及格 |
|------|------|------|------|--------|
| 响应时间 | <1s | 1-3s | 3-5s | >5s |
| 错误率 | <0.1% | 0.1-1% | 1-5% | >5% |
| CPU使用率 | <60% | 60-70% | 70-80% | >80% |
| 内存使用率 | <60% | 60-70% | 70-80% | >80% |

## 3. 常用性能测试工具

### 工具对比

| 工具 | 类型 | 语言 | 特点 | 适用场景 |
|------|------|------|------|----------|
| JMeter | 开源 | Java | 功能全面，插件丰富 | Web、API测试 |
| Locust | 开源 | Python | 脚本灵活，分布式 | API、自定义协议 |
| LoadRunner | 商业 | C | 企业级，协议广泛 | 大型企业项目 |
| Gatling | 开源 | Scala | 高性能，报告详细 | 高并发场景 |
| k6 | 开源 | JavaScript | 现代化，CI友好 | DevOps集成 |

### JMeter 简介

```
JMeter 核心组件
├── 线程组 (Thread Group)
│   ├── 线程数: 模拟的用户数
│   ├── Ramp-Up: 启动时间
│   └── 循环次数: 执行次数
├── 取样器 (Sampler)
│   ├── HTTP Request
│   ├── JDBC Request
│   └── FTP Request
├── 监听器 (Listener)
│   ├── Summary Report
│   ├── Aggregate Report
│   └── View Results Tree
├── 断言 (Assertion)
│   ├── Response Assertion
│   └── JSON Assertion
├── 配置元件 (Config Element)
│   ├── CSV Data Set Config
│   └── HTTP Header Manager
└── 定时器 (Timer)
    ├── Constant Timer
    └── Gaussian Random Timer
```

### JMeter 基础脚本

```xml
<?xml version="1.0" encoding="UTF-8"?>
<jmeterTestPlan version="1.2">
  <hashTree>
    <TestPlan guiclass="TestPlanGui" testclass="TestPlan" testname="性能测试计划">
      <elementProp name="TestPlan.user_defined_variables" elementType="Arguments">
        <collectionProp name="Arguments.arguments"/>
      </elementProp>
    </TestPlan>
    <hashTree>
      <ThreadGroup guiclass="ThreadGroupGui" testclass="ThreadGroup" testname="线程组">
        <intProp name="ThreadGroup.num_threads">100</intProp>
        <intProp name="ThreadGroup.ramp_time">60</intProp>
        <boolProp name="ThreadGroup.scheduler">true</boolProp>
        <stringProp name="ThreadGroup.duration">300</stringProp>
      </ThreadGroup>
    </hashTree>
  </hashTree>
</jmeterTestPlan>
```

## 4. 性能测试监控

### 服务器监控

```bash
# CPU监控
top
vmstat 1
mpstat -P ALL 1

# 内存监控
free -h
vmstat 1

# 磁盘IO监控
iostat -x 1
iotop

# 网络监控
iftop
netstat -an

# 综合监控
nmon
```


## 5. 性能瓶颈分析

### 常见瓶颈

```
性能瓶颈
├── CPU瓶颈
│   ├── 症状: CPU使用率持续高于80%
│   ├── 原因: 计算密集、死循环、频繁GC
│   └── 优化: 算法优化、缓存、异步处理
├── 内存瓶颈
│   ├── 症状: 内存持续增长、频繁GC
│   ├── 原因: 内存泄漏、大对象、缓存过大
│   └── 优化: 修复泄漏、调整堆大小、优化缓存
├── IO瓶颈
│   ├── 症状: 磁盘IO高、等待时间长
│   ├── 原因: 频繁读写、同步阻塞
│   └── 优化: 异步IO、缓存、SSD
├── 网络瓶颈
│   ├── 症状: 网络延迟高、带宽满
│   ├── 原因: 数据量大、网络抖动
│   └── 优化: 压缩、CDN、连接池
└── 数据库瓶颈
    ├── 症状: 慢查询、连接池满
    ├── 原因: 索引缺失、SQL低效
    └── 优化: 索引优化、读写分离、分库分表
```

### 性能调优策略

| 层面 | 策略 |
|------|------|
| 代码层 | 算法优化、减少循环、避免重复计算 |
| 架构层 | 缓存、异步、微服务、负载均衡 |
| 数据库层 | 索引优化、SQL优化、读写分离 |
| 系统层 | 内核参数调优、文件描述符、连接数 |
| 网络层 | 压缩、CDN、HTTP/2 |
