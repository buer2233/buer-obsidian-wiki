# 测试平台基础

## 1. 测试平台的价值

### 为什么需要测试平台

| 痛点 | 平台解决方案 |
|------|-------------|
| 用例分散在Excel/文档中 | 统一管理，结构化存储 |
| 执行记录无法追溯 | 自动记录执行历史 |
| 测试报告手动编写 | 自动生成报告 |
| 环境配置复杂 | 一键切换环境 |
| 团队协作困难 | 权限管理，协作执行 |
| 回归测试耗时 | 定时任务，自动触发 |

### 核心价值

```
效率提升
├── 减少重复劳动
├── 提高用例复用率
└── 缩短回归周期

质量保障
├── 测试覆盖率可视化
├── 缺陷趋势分析
└── 质量门禁

流程规范
├── 标准化测试流程
├── 测试左移
└── 持续集成
```

## 4. 数据库设计

### 核心表结构

```sql
-- 项目表
CREATE TABLE projects (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    owner_id INT,
    is_active BOOLEAN DEFAULT TRUE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 测试用例表
CREATE TABLE test_cases (
    id INT PRIMARY KEY AUTO_INCREMENT,
    project_id INT NOT NULL,
    module_id INT,
    title VARCHAR(200) NOT NULL,
    priority ENUM('P0', 'P1', 'P2', 'P3') DEFAULT 'P2',
    precondition TEXT,
    steps TEXT,
    expected_result TEXT,
    created_by INT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 测试执行表
CREATE TABLE test_executions (
    id INT PRIMARY KEY AUTO_INCREMENT,
    case_id INT NOT NULL,
    plan_id INT,
    status ENUM('pass', 'fail', 'block', 'skip') NOT NULL,
    executor_id INT,
    executed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    duration INT,
    remark TEXT,
    FOREIGN KEY (case_id) REFERENCES test_cases(id)
);

-- 缺陷表
CREATE TABLE bugs (
    id INT PRIMARY KEY AUTO_INCREMENT,
    project_id INT NOT NULL,
    title VARCHAR(200) NOT NULL,
    severity ENUM('critical', 'major', 'minor', 'trivial'),
    status ENUM('open', 'in_progress', 'resolved', 'closed'),
    reporter_id INT,
    assignee_id INT,
    steps_to_reproduce TEXT,
    expected_result TEXT,
    actual_result TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

## 5. API 设计规范

### RESTful 风格

| 操作 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 查询列表 | GET | /api/projects | 获取项目列表 |
| 查询详情 | GET | /api/projects/:id | 获取单个项目 |
| 创建 | POST | /api/projects | 创建项目 |
| 更新 | PUT | /api/projects/:id | 更新项目 |
| 删除 | DELETE | /api/projects/:id | 删除项目 |
| 搜索 | GET | /api/projects?name=test | 搜索项目 |

### 响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "name": "测试项目"
  }
}

{
  "code": 400,
  "message": "参数错误",
  "errors": [
    { "field": "name", "message": "名称不能为空" }
  ]
}

{
  "code": 200,
  "message": "success",
  "data": {
    "total": 100,
    "page": 1,
    "size": 10,
    "items": [...]
  }
```

## 6. 部署方案

### Docker 部署

```yaml
# docker-compose.yml
version: '3.8'

services:
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=mysql://user:pass@db:3306/testplatform
      - REDIS_URL=redis://redis:6379/0
    depends_on:
      - db
      - redis

  db:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=rootpass
      - MYSQL_DATABASE=testplatform
    volumes:
      - mysql_data:/var/lib/mysql

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  celery:
    build: ./backend
    command: celery -A config worker -l info
    depends_on:
      - db
      - redis

volumes:
  mysql_data:
```

### 启动命令

```bash
# 开发环境
docker-compose up -d

# 生产环境
docker-compose -f docker-compose.prod.yml up -d

# 查看日志
docker-compose logs -f backend
```
