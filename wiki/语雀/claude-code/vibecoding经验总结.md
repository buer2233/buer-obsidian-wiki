## 针对开发skills的总结

### 明确 Agent 的边界

**1. Agent 要完成什么任务？**
编写接口自动化用例

**2. 不允许做什么？**
不允许修改指定测试用例和测试方法之外的文件

**3. 哪些操作必须人工审批？**
抓包后的接口数据需要人工审核，包括是否新增接口和已覆盖接口是否使用到当前接口自动化用例中

**4. 失败时怎么办？**
包括执行前用户给到的提示词不合规的错误，和执行过程中的读取参数错误，开启抓包服务的错误，最后执行pytest闭环时检测到为环境问题或其它非用例导致的执行失败错误等。

**5. 成功如何定义？**
按用户要求新增接口自动化用例后，成功执行完成pytest闭环。

## AI编程的流程和技巧总结

### 需求分析

做项目的第一步不是让 AI 写代码，而是要先想清楚做什么。明确项目目标和核心功能，将大需求拆解为小任务，确定技术栈和项目架构。

早期做项目的时候，恨不得一条提示词把整个页面全交给 AI 一把梭，结果生成出来的代码问题一大堆。后来学会了「小步快跑」，一次只让 AI 完成一个小功能，做完验证没问题再继续下一个，效率反而高了很多。

### 提示词工程

提示词工程是 AI 编程的核心技能，写好提示词能让 AI 生成的代码质量提高好几个档次。

一个好的提示词通常包含角色设定、背景信息、具体需求、约束条件和输出格式。

**万用技巧：**

1. **需求描述要具体，不要模棱两可** - 比如「做得好看」应该改成「背景改成蓝色渐变，按钮加圆角」

2. **遇到复杂任务时，要进行拆解**

3. **给 AI 举例子，它会更容易理解**

4. **学会不断追问和迭代**

### 上下文管理

光靠提示词还不够，还要学会给 AI 提供充足的参考信息。

最重要的是学会使用项目规则文件。建议优先学习通用的 CLAUDE.md 和 AGENTS.md 格式，几乎所有主流 AI 编程工具都支持。在规则文件里写入项目技术栈、编码规范、目录结构说明，AI 生成代码时就会自动遵守这些约定。

**各工具默认读取的规则文件：**

| 工具 | 默认读取 |
|------|---------|
| Claude Code | 主要读取 CLAUDE.md |
| Codex | 主要读取 AGENTS.md |
| Claude Code 是否自动读 AGENTS.md | 通常不会，除非在 CLAUDE.md 里用 @AGENTS.md 显式导入 |
| Codex 是否自动读 CLAUDE.md | 默认不会；只有在配置 fallback 且当前目录没有 AGENTS.md 时才会读 |

**多工具协作方案：**

当项目中同时有 CLAUDE.md 和 AGENTS.md 时，它们不是天然冲突关系，而是两个工具各自的入口文件。

如果同时使用 Claude Code、Codex、Cursor 等多个工具，可以把 AGENTS.md 作为共通规则，然后在 CLAUDE.md 中用 @AGENTS.md 读取，再追加 Claude Code 特有规则。这样就可以实现仅维护一个文件。

### Bug 排查

无论是人还是 AI，都会遇到 Bug，所以要学会排查和解决 Bug。

**向 AI 描述 Bug 时：**
- 告诉它期望的效果和实际的效果
- 提供完整的错误信息
- 说明已尝试的解决方案

**遇到解决不了的问题：**
- 及时回退到上一个稳定版本，不要让 AI 一条路走到黑
- 如果发现 AI 卡在同一个问题上反复修不好，试试换个角度描述问题
