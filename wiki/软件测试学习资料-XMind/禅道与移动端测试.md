# 禅道.git.移动端测试

## git

### 定义

#### git是一个分布式的版本控制系统

### 集中式和分布式的区别

#### 集中式
- 必须有一个中央服务器和网络才能工作

#### 分布式
- 可以不用中央服务器和网络也可以工作
- 但实际工作,还是会需要有一台中央服务器,仅仅是用于便便交换修改

### 查看文件状态

#### ?
- 新建文件,还不受版本控制

#### +
- 将一个文件准备加入版本控制

#### !
- 文件已修改,未提交/推送

#### √
- 已提交/推送

### 常用Git命令

#### 克隆
- git clone 远程仓库地址

#### 初始化
- git init

#### 提交
- 将修改后的文件提交到暂存区
- git add 文件名
- 将暂存区的文件提交到本地仓库
- git commit -m "日志信息"

#### 推送
- git push origin 远程地址

#### 拉取
- git pull

什么时候会产生冲突?

	当2个用户同时修改同一个文件,其中一个用户先提交并推送,后提交并推送的这个用户就会产生冲突,因为其本地版本库与远程版本库的内容已经不一样了!



如何避免冲突?

	每次修改文件之前一定要先拉取,修改完成后及时的提交与推送.



编辑冲突=>解决冲突=>提交=>推送

- 是
- 编辑/解决冲突=>重新推送
- 否
- 查找失败原因并解决=>重新推送

## 缺陷管理工具-禅道

### 1.禅道软件的基本介绍

#### 国产开源项目管理软件

#### 青岛易软天创网络科技有限公司开发

#### 禅道：名字是受《编程之道》和《编程之禅》这两本书的启发

#### 设计理念：基于国际流行的敏捷项目管理方法—Scrum

#### 优点
- 功能全
- 规范
- 开源
- 美观、易用
- 安装、配置简单
- 免费
- 布置灵活

### 禅道软件的安装与配置

#### 32位与64位操作系统的区别
- 32位操作系统：最多支持4G的内存，实际内存为3.25G；
- 64位操作系统：支持4G 8G 16G 32G 64G 128G 256G内存，理论上可以无限支持，只要你主板上有足够的内存条。

#### 官网下载安装包：安装在C、D盘根目录

#### 修改数据库密码

#### 登录禅道数据库
- 服务器：127.0.0.1:3306

用户名：root

密码默认为：123456

数据库名：zentao

#### 登录禅道
- 默认用户名：admin; 密码：123456

### 创建分组和用户

#### 以管理员身份登录

#### 用户分组和部门结构的区别
- 部门结构是公司从组织角度来讲的一个划分，它决定了公司内部人员的上下级汇报关系
- 用户分组主要用来区分用户权限

#### 受限用户分组
- 属于该分组的用户在禅道里 只能编辑与自己相关的 需求、任务、Bug等 ，不能新增 需求、任务、Bug等；
- 针对禅道整个系统的使用受限

#### 团队管理-受限用户
- 是只针对某一个项目的受限操作，其他项目不受影响。

#### 权限维护的注意事项
- 一个用户在多个权限分组里，其在禅道里的权限取的是各个权限分组里权限的合集
- 要访问到一个Bug，必须同时拥有Bug所在产品，项目的访问权限和Bug详情的权限。

## 移动端测试

### adb

#### 全称Android Debug Bridge, 是android sdk里的一个工具,目的是起到调试桥的作用，用这个工具可以直接操作管理android模拟器或者真实的andriod设备(手机)，是客户端测试常用的辅助工具。

### adb常用命令

#### 获取设备列表及设备状态
- adb devices
- 雷电模拟器的默认自动连接
- adb connect ip:端口
- 连接真机

#### 结束和启动命令
- adb kill-server
- 结束命令
- adb start-server
- 启动命令

#### 抓取安卓日志
- adb logcat -c
- 清除之前的日志信息
- adb logcat
- 打印默认日志数据
- adb logcat -v time
- 需要打印日志详细时间的简单数据
- adb logcat -v time > D:\log.txt
- 将日志保存到电脑固定的位置，比如D:\log.txt

#### 将安卓设备的文件复制到本地电脑
- adb pull 源文件路径 目标路径

#### 将本地电脑的文件复制到安卓设备
- adb push 源文件路径 目标路径

#### 安装应用
- adb install *.apk

#### 卸载软件
- adn uninstall <包名>

#### 列出应用包名
- adb shell pm list package | findstr “taobao”

#### 获取root权限
- adb root

#### Monkey压力测试
- adb shell monkey -p <packagename> -v 500(测试次数)
