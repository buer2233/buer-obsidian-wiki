# MySQL总结

## 数据库基础知识

### 什么是数据库

#### 数据库是按照一定的数据结构,进行组织,存储,管理数据的仓库,是一种对大量信息进行管理的高效解决方案.

#### 数据库系统DBS
- 数据库DB
- 数据库管理系统DBMS

### 数据库的分类

#### 关系型
- Oracl,MySQL,SQL Server...
- 建立在关系模型上的数据库,关系模型就是二维表模型.其中的二维表记录着实体信息和实体之间的关系信息.

#### 非关系型
- MongoDB,Redis...
- 不建立在关系模型上的数据库,数据结构不够严谨

### 数据库的常用语言

#### 数据库系统DBS
- 数据库DB
- 表table
- 行row
- 记录record
- 列column
- 字段field
- 数据库管理系统DBMS

#### 结构化查询语言SQL

### 管理数据库

## 数据类型

### 字段类型

#### 数字型
- 整数型
- tinyint
- 占用一个字节,一共能表示256个数
- 有符号:-128~127
- 无符号:0~255
- int
- 占用4个字节
- 有符号:-21亿~21亿
- 无字符:0~42亿
- 使用unsigned限制为无字符,否则默认为有字符
- 小数型
- 浮点数
- float(M,D)
- 单精度浮点数
- double(M,D)
- 双精度浮点数
- 定点数
- decimal(M,D)
- 数据不会有损失,常用于记录货币.

#### 文本型
- char(M)
- 定长字符,M表示最大字符数.优势为运算速度快
- 常用在255个字符以内固定长度的字符.比如身份证,电话号码等.
- varchar(M)
- 变长字符,M表示最大的字符数.优势为节省空间.
- 常用在255个字符以内,长度不确定的字符.
- text
- 常用在256个字符以上的文本中

#### 日期时间型
- datetime
- 固定的日期时间
- timestamp
- 时间戳:当新增记录或更新记录时自动更新为当前系统时间,用于记录最后一次修改的时间或新插入记录的时间

### 字段属性

#### not null 不能为空
- 设置该字段的值不能为空,不写就是默认可以为空

#### default 默认值
- 设置一个默认值,没有数据传入就是使用默认值.有数据传入则使用传入的值

#### comment 备注
- 用汉字备注字段,方便后期维护

#### unique key 唯一值
- 该字段的值不能重复,但可以为空

#### primary key 主键
- 用于唯一标识一条记录
- 一个表最能只能有一个主键
- 不能为空
- 不能重复

#### auto_increment 自动增长
- 在新插入记录时,自动在本字段最大值的基础上加1,条件本字段的类型必须为整数型.常与主键一起使用,但不是必须与主键一起使用.

## DQL(数据查询语言)

### select [选项] 字段1 as 别名1,...字段N as 别名N from `表名` where 条件表达式;

### select 选项

#### all:所有
- 默认不写就是所有

#### as:别名
- 将查询后的字段名用别名显示,方便查询.可以省略as,直接在字段后写别名.

#### distinct:去重
- 重复的定义:查询后的数据里,同一条记录的所有字段都相同,才认定为重复

#### 常见聚合函数
- count()计数
- Max()最大值
- Min()最小值
- sum()求和
- avg()平均值
- group_concat
- 将分组后的同组字段显示全部显示在一起

### from 连表查询join

#### 内连接:inner join
- 将参与连接的两个表中符合连接条件的记录查询出来,不符合的过滤掉.
- 连接的字段名不同时
- select `字段` from `表1` inner join `表2` on `表1的连接字段` = `表2的连接字段`;  注:inner join可只写 join .
- 连接的字段名相同时
- select `字段` from `表1` inner join `表2` using(`连接的字段名`);

#### 左外连接:left join
- 将参与连接的左表中即使不能匹配连接条件的记录也会查询出来.右表不符合的过滤掉
- select 字段 from `表1` left join `表2` on/using `连接字段`

#### 右外连接:right join
- 将参与连接的右表中即使不能匹配连接条件的记录也会查询出来.左表不符合的过滤掉
- select 字段 from `表1` right join `表2` on/using `连接字段`

### where 条件表达式

#### 逻辑表达式
- and  or  not

#### 比较运算符
- +  -  *  /  <  >  =  !=

#### in / not in
- 判断字段在不在字段的集合中

#### between 数字1 and 数字2
- 字段在数字1和数字2之间

#### is null

#### like
- 模糊判断

### group by

#### 对数据进行分组, 并统计

#### group by单独使用, 默认查出每组数据的第一条

#### group by通常和聚合函数一起使用, 起到分组统计的作用

#### group by 字段1，字段2
- 先按照字段1分组再按字段2分组

### having

#### 对分组后的数据, 进行再次筛选. 因此位于where和group by之后

### order by

#### 默认升序排序, 其后加的desc为降序排序

#### 多条件排序
- 先排序前一个字段, 在前一个字段的基础上再排序下一个字段

### limit(记录开始, 记录结束)

#### 限制显示输出的数据条数

#### 记录开始的索引默认从0开始

## 子查询

### 一条SQL语句的查询结果,作为另一个SQL语句的条件

### 按位置分类

#### 1.where子查询
- 子查询的SQL在where里面

#### 2.from子查询
- 子查询的SQL在from里面

### 按结果分类

#### 1.表子查询
- 子查询的结果为一个表
- 通过子查询获取一张新表,被另一个SQL通过in进行过滤(where过滤条件的字段需和新表的字段一致)

#### 2.列子查询
- 子查询的数据为一列数据
- 通过子查询获取一列数据,被其它SQL通过in进行过滤

## 外键约束

### 定义

#### 指有关联的两个数据表之间的跨表条件约束,从表数据依赖于主表数据

#### 主表
- 关联的字段为主键的表

#### 从表
- 关联的字段为普通字段的表

### 注意

#### 1. 外键约束要求表的引擎为innoDB

#### 2. 外键约束的主表和从表的关联字段,数据类型严格一致

### 作用

#### 外键约束可以保证主表和从表数据的合理性,限制不合理的数据添加和修改,删除

### 外键约束的级别

#### 严格级别(restrict)
- 默认的级别, 不允许不对称的修改

#### 级联级别(cascade)
- 主表变化,从表对应变化

#### 置空操作(set null)
- 主表变化, 从表变为空

## 实体之间的关系

### 一对一

#### 通过相同的主键实现

### 一对多

#### 在多的一段添加字段保存,少的一端的主键

### 多对多

#### 通过创建中间表的方式实现
- 中间表取两个表的主键, 但中间表没有加主键

## 事务

### 事务是一段连续的,不可分割的SQL语句(一组SQL),要么全部成功后提交事务,要么有一个失败就全部失败,回滚.

### 目的

#### 保证MySQL数据库中的数据的完整性

### 前提

#### 数据表的引擎必须是innoDB

### 步骤

#### 1.开启事务(begin, start transaction)
- 设置一组SQL为事务

#### 2.1 提交事务(commit)
- 在内存中执行的所有SQL都成功, 并修改硬盘中的数据库数据

#### 2.2 回滚(roll back)
- 在内存中执行的SQL,有任何一条错误时,都会执行回滚,清空内存中的所有SQL语句,不存储数据到硬盘

### 特点

#### 原子性(Atomicty)

#### 移植性(Consistency)

#### 隔离性(Isolation )

#### 持久性(Durability)
